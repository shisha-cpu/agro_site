# 🚀 Запуск сервера

## Вариант 1: Быстрый старт (данные из файла)

Если MongoDB не подключается, можно запустить сервер в режиме загрузки данных из файла:

```bash
# Сервер запустится и будет загружать данные из data.json при каждом запросе
npm start
```

## Вариант 2: Полноценный сервер с MongoDB

### 1. Проверьте подключение к MongoDB

Убедитесь, что:
- Есть подключение к интернету
- MongoDB URI правильный в файле `.env`
- MongoDB Atlas доступен из вашей сети

### 2. Импорт данных

После запуска сервера:

```bash
# Способ А: Через админ-панель
# 1. Откройте http://localhost:3000/admin.html
# 2. Нажмите "📥 Импорт из data.json"

# Способ Б: Через команду
npm run seed
```

## 🔧 Решение проблем

### Ошибка: Operation `products.countDocuments()` buffering timed out

Это означает, что сервер не может подключиться к MongoDB.

**Варианты решения:**

1. **Проверьте MongoDB URI** в файле `.env`:
   ```env
   MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/DatabaseName
   ```

2. **Проверьте доступность MongoDB Atlas**:
   - Убедитесь, что ваш IP адрес добавлен в whitelist MongoDB Atlas
   - Проверьте, что пароль не содержит специальных символов (или закодируйте их)

3. **Временное решение** - используйте локальные данные:
   - Откройте `js/data.js`
   - Закомментируйте загрузку с API
   - Раскомментируйте загрузку из data.json

### Ошибка: Network Error

- Проверьте подключение к интернету
- Попробуйте перезапустить сервер

## 📝 Команды

```bash
# Запуск сервера
npm start

# Запуск в режиме разработки (nodemon)
npm run dev

# Импорт данных из data.json в MongoDB
npm run seed
```

## ✅ Проверка работы

1. Откройте http://localhost:3000
2. Проверьте API: http://localhost:3000/api/stats
3. Админ-панель: http://localhost:3000/admin.html
